အခန်း ၇ – Branches (ခွဲခြားလိုင်းများ)

Git ရဲ့ အင်အားကြီးတဲ့ လုပ်ဆောင်ချက်တွေထဲမှာတစ်ခုကတော့ branching ဖြစ်ပါတယ်။ Git ဟာ ခွဲခြားလိုင်းတွေကို ပြုလုပ်ဖို့အလွယ်တကူလုပ်နိုင်ပြီး၊ လျင်မြန်ပြီး သက်သာစွာ switch လုပ်နိုင်တယ်။ Git မှာ branch တစ်ခုဖန်တီးဖို့ဆိုတာဟာ commit ကို ပြောင်းဖို့လိုသလောက်ပင်လွယ်ကူပါတယ်။

Git ဟာ development ရဲ့ ခွဲခြားလိုင်းတစ်ခုချင်းစီကို branch တစ်ခုအဖြစ် ကိုယ်စားပြုပါတယ်။ ဒီ branching mechanism ဟာ သာမန် version control systems တွေထက် ကောင်းမွန်ပြီး flexible ဖြစ်တယ်။ Branch တွေကြောင့် feature development, bug fixes, experimentation စတာတွေကို အဓိကလိုင်း (mainline) ကို မထိခိုက်ဘဲ လုပ်ဆောင်နိုင်ပါတယ်။

Branch တစ်ခုတည်ဆောက်ရင် Git ဟာ pointer တစ်ခုကို ဖန်တီးပါတယ်။ အဲ့ဒီ pointer ကတော့ commit တစ်ခုကိုညွှန်နေပြီး သူ့နောက်မှာဖြစ်လာမယ့် commits တွေကို ဆက်ခံနေတာဖြစ်ပါတယ်။

ဒီအခန်းမှာတော့ Git မှာ branch တည်ဆောက်ခြင်း၊ ပြောင်းခြင်း၊ ရှာဖွေခြင်းနဲ့ ပတ်သက်တဲ့ commands တွေကို လေ့လာသွားမှာဖြစ်ပါတယ်။
