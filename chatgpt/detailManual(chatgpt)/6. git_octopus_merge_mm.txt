🐙 Git Octopus Merge အကြောင်း (မြန်မာဘာသာ)

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🧠 Octopus Merge ဆိုတာဘာလဲ?
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Git မှာ "Octopus Merge" ဆိုတာက 
➡️ သုံးခု (သို့မဟုတ်) သုံးခုထက်ပိုတဲ့ branches တွေကို 
➡️ တချိန်တည်းမှာ တစ်ခုတည်းသော merge commit အနေနဲ့ ပေါင်းထည့်တဲ့နည်းပညာတစ်ခုပဲဖြစ်ပါတယ်။

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📌 အသုံးဝင်တဲ့အချိန်
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
- များများသော feature branches တွေကို တပြိုင်နက်တည်း merge ချင်တဲ့အခါ
- Conflict မဖြစ်တဲ့ changes များကို ပေါင်းချင်တဲ့အခါ
- Automation scripts တွေမှာ တစ်ခါတည်း အများစု merge လုပ်ချင်တဲ့အခါ

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🛠️ ဥပမာ
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
$ git checkout main
$ git merge branch1 branch2 branch3

➡️ အထက်ပါ command က branch ၃ ခုကို တစ်ခါတည်း main ထဲ merge လုပ်တယ်။
➡️ Git က merge commit တစ်ခုတည်း ဖန်တီးပြီး branches ၃ ခုရဲ့ change တွေကို ပေါင်းတယ်။

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
❗ သတိပြုရန်
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
- Git ဟာ Octopus Merge မှာ **conflict ဖြစ်လာရင် ဖြေရှင်းပေးမှာ မဟုတ်ပါ။**
- Conflict ဖြစ်မယ်ဆိုရင် Git က merge fail ဖြစ်သွားပါတယ်။
- ဒါကြောင့် Octopus Merge ကို **အလွယ်တကူပေါင်းနိုင်တဲ့ commits** များအတွက်သာ သုံးသင့်ပါတယ်။

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
✅ အကျဉ်းချုပ်
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
- 3+ branches ကို တစ်ခါတည်း merge လုပ်တတ်တဲ့နည်း
- Merge commit တစ်ခုတည်းဖန်တီးတယ်
- Conflict မဖြစ်တဲ့အခါမှာသာအဆင်ပြေ
- Automation များအတွက်အသုံးဝင်တယ်

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
သတိ – Conflict ရှိမယ်ထင်ရင် သာမန် 3-way merge သုံးသင့်ပါတယ်။
